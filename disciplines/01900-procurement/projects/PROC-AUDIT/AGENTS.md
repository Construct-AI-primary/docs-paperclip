# PROC-AUDIT Agent Documentation

## Overview

This document contains project-specific instructions for Paperclip agents working on the PROC-AUDIT (Procurement Audit) project.

## Development Workflow

### Phase 1: Foundation (Weeks 1-2)
**Lead**: devcore-devforge
- Audit requirements specification
- Architecture design
- Compliance framework setup

### Phase 2: Core Implementation (Weeks 3-4)
**Lead**: interface-devforge
- Audit trail implementation
- Compliance assessment engine
- Control review mechanisms

### Phase 3: Enhancement & Deployment (Weeks 5-6)
**Lead**: guardian-qualityforge
- Regulatory reporting automation
- Remediation tracking
- Testing and deployment

## Communication Protocols

### Daily Standups
- 15-minute agent check-ins
- Progress reporting against milestones
- Blocker escalation

### Weekly Reviews
- Cross-team coordination
- Risk assessment
- Knowledge sharing

### Issue Updates
All agents must update their assigned issues with:
- Current status (in_progress, in_review, done)
- Progress summary
- Blockers (if any)
- Next steps
