# LOGISTICS-PLATFORM Shared Database Schema Reference

## Core Tables

### logistics_records
| Column | Type | Description |
|--------|------|-------------|
| id | UUID | Primary key, autogenerated `01700-{YYYY}-{NNNNN}` |
| company_id | UUID | FK to companies table |
| logistics_area | TEXT | One of: container_tracking, customs, delivery, documents, shipping, site, traffic, materials |
| status | TEXT | draft, submitted, approved, rejected |
| created_at | TIMESTAMPTZ | Auto-set |
| updated_at | TIMESTAMPTZ | Auto-set |
| created_by | UUID | FK to users |

### logistics_details
| Column | Type | Description |
|--------|------|-------------|
| id | UUID | Primary key |
| record_id | UUID | FK to logistics_records |
| detail_data | JSONB | Area-specific data payload |
| status | TEXT | draft → submitted → approved workflow |

### logistics_audit_log
| Column | Type | Description |
|--------|------|-------------|
| id | UUID | Primary key |
| record_id | UUID | FK to logistics_records |
| event_type | TEXT | CREATE_, UPDATE_, DELETE_, APPROVE_ |
| changed_by | UUID | FK to users |
| changed_at | TIMESTAMPTZ | Auto-set |

## Indexes
- Composite: (company_id, status, created_at) on logistics_records
- Composite: (company_id, logistics_area) on logistics_records

## Partitioning
- Monthly partitions: logistics_records_y2026m04, logistics_records_y2026m05, etc.
- Auto-creation function for next 6 months

## RLS Policies
- Company-level isolation: users see only their company's records
- Role-based: admin=CRUD, dispatcher=CRUD, driver=read+update_status, customs_agent=read+update_customs, site_manager=CRUD_site, viewer=read

## Caching
- Redis key scheme: `logistics:{company_id}:{record_id}`
- Default TTL: 24 hours
- Event-driven invalidation on INSERT/UPDATE/DELETE
