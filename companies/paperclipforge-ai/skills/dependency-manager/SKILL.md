# Dependency Manager - PaperclipForge AI Dependency Resolution Specialist

## Overview
Dependency Manager specializes in dependency tracking, resolution planning, and critical path analysis within the PaperclipForge AI operational ecosystem. Dependency Manager identifies, tracks, and resolves complex interdependencies between issues, ensuring efficient project execution and preventing bottlenecks.

## When to Use
- When dependency tracking between issues is needed
- When resolution planning for complex dependencies is required
- When critical path analysis and optimization is needed
- When issue interdependency management is required
- When dependency conflict resolution is needed
- **Don't use when:** Issue assignment is needed (use assignment-specialist), or progress tracking is needed (use progress-tracker)

## Core Procedures

### Dependency Tracking Workflow
1. **Dependency Identification** - Identify dependencies between issues and projects
2. **Dependency Mapping** - Map dependency relationships and hierarchies
3. **Impact Analysis** - Analyze dependency impacts on timelines and resources
4. **Critical Path Identification** - Identify critical paths and bottleneck dependencies
5. **Dependency Documentation** - Document all identified dependencies and relationships

### Resolution Planning Workflow
1. **Dependency Prioritization** - Prioritize dependencies based on impact and urgency
2. **Resolution Strategy Development** - Develop strategies for resolving complex dependencies
3. **Resource Planning** - Plan resources needed for dependency resolution
4. **Timeline Optimization** - Optimize timelines considering dependency constraints
5. **Contingency Planning** - Develop contingency plans for dependency risks

### Critical Path Analysis Workflow
1. **Path Identification** - Identify all possible paths through dependency networks
2. **Critical Path Calculation** - Calculate critical paths and float times
3. **Bottleneck Analysis** - Analyze bottlenecks and constraint points
4. **Optimization Opportunities** - Identify opportunities for path optimization
5. **Risk Assessment** - Assess risks associated with critical path dependencies

### Conflict Resolution Sub-Workflow
1. **Conflict Identification** - Identify dependency conflicts and resource constraints
2. **Conflict Analysis** - Analyze root causes of dependency conflicts
3. **Resolution Planning** - Develop plans to resolve identified conflicts
4. **Stakeholder Coordination** - Coordinate with stakeholders affected by conflicts
5. **Implementation Monitoring** - Monitor implementation of conflict resolutions

## Dependency Management Scope
- **Dependency Tracking** - Identification and mapping of issue interdependencies
- **Resolution Planning** - Strategy development for complex dependency resolution
- **Critical Path Analysis** - Path optimization and bottleneck identification
- **Conflict Resolution** - Dependency conflict identification and resolution
- **Risk Management** - Dependency-related risk assessment and mitigation

### Cross-Company Dependency Integration
- **Issue Generator:** Receive dependency information during issue creation
- **Assignment Specialist:** Consider dependencies during intelligent routing
- **Progress Tracker:** Monitor dependency resolution progress
- **Operations Director:** Receive dependency analytics and critical path reports
- **DevForge AI (Nexus):** Coordinate technical dependency resolution

- **Error:** Dependency not identified
  **Response:** Conduct dependency audit and update identification processes
- **Error:** Critical path miscalculation
  **Response:** Recalculate critical paths and adjust project plans
- **Error:** Dependency conflict unresolved
  **Response:** Escalate to operations director and implement emergency resolution
