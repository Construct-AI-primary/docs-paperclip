# Knowledge-Flow & Docs Team

Team focused on tracking knowledge flow from field data to VFS to lessons to users, and maintaining documentation hygiene.

## Team Lead

- `learningforge-ai-knowledge-flow-division-lead`

## Team Members

| Agent | Role | Focus |
|-------|------|-------|
| `learningforge-ai-vfs-markdown-tracker` | File Operation Logging | Log every write/update to Markdown files in VFS |
| `learningforge-ai-knowledge-provenance` | Trace Graph Building | Build trace graph from findings to lessons |
| `learningforge-ai-doc-usage-analyzer` | Usage Analytics | Measure how often each Markdown file is used |
| `learningforge-ai-from-doc-to-lesson-generator` | Content Transformation | Generate lessons from updated Markdown files |
| `learningforge-ai-doc-gap-detector` | Gap Identification | Compare field patterns with current docs |
| `learningforge-ai-workflow-coach` | Workflow Enforcement | Nudge users/agents to follow workflow |
| `learningforge-ai-knowledge-transparency` | Knowledge Trail Display | Generate knowledge-trail notes for users |
| `learningforge-ai-knowledge-hygiene` | Documentation Maintenance | Scan VFS for stale/conflicting docs |
| `learningforge-ai-audit-by-path` | Audit Trail Indexing | Index complete chain from findings to lessons |

## Team Goals

1. Log every write/update to Markdown files in VFS with metadata
2. Build trace graph from findings → notes → templates → lessons
3. Measure how often each Markdown file is used and flag orphaned docs
4. Generate lessons from updated or newly-created Markdown files
5. Compare field patterns with current docs and propose new templates
6. Nudge users/agents to follow: defect → note → template → checklist → lesson
7. Scan VFS for stale/conflicting docs and propose cleanup monthly

## Integration with Other Teams

- **Learning & Curriculum**: Provide document analysis and lesson content
- **Safety & Quality**: Track defect and safety documentation
- **Research & Compliance**: Monitor AutoResearch output documentation
