### Template Selection Decision Framework

#### Primary Decision Framework

The agents use a **multi-layered decision tree** to select templates:

##### Layer 1: Workflow Complexity Analysis
```
Input: Workflow metadata (stakeholders, timeline, deliverables, constraints)
├── High Complexity (>20 stakeholders, >6 months, >50 deliverables)
│   └── Use: Multi-Agent Orchestration Template
├── Medium Complexity (5-20 stakeholders, 1-6 months, 10-50 deliverables)
│   └── Use: Standard Workflow Template + customization
└── Low Complexity (<5 stakeholders, <1 month, <10 deliverables)
    └── Use: Simple Task Template
```

##### Layer 2: Domain-Specific Pattern Matching
```
Input: Discipline code, project type, regulatory requirements
├── Engineering Disciplines (00800-00899 series)
│   ├── Design Phase: Specification Development Template
│   ├── Construction Phase: Construction Admin Template
│   └── Closeout Phase: Commissioning Template
├── Procurement Disciplines (01900 series)
│   ├── Transaction Heavy: Procurement Order Template
│   └── Compliance Heavy: Regulatory Compliance Template
└── Safety Disciplines (02400 series)
    └── Risk Assessment: Safety & Risk Management Template
```

##### Layer 3: Resource & Timeline Optimization
```
Input: Available agents, skill coverage, timeline constraints
├── Resource Rich: Use advanced templates with parallel processing
├── Time Critical: Use pre-optimized templates with accelerated workflows
├── Skill Gaps: Use templates with built-in training/onboarding components
└── Multi-Platform: Use cross-platform templates with integration requirements
```

#### Template Selection Algorithm

The **Workflow Complexity Analyzer** (DevForge AI) uses this algorithm:

1. **Parse Workflow Metadata**
   - Extract stakeholders, deliverables, constraints, timeline
   - Calculate complexity score (0-100)

2. **Pattern Recognition**
   - Match against 23 discipline workflow catalogs
   - Identify common patterns across disciplines
   - Calculate template fit scores

3. **Resource Assessment**
   - Check agent availability and skills
   - Evaluate timeline feasibility
   - Assess integration requirements

4. **Template Ranking & Selection**
   - Rank templates by reusability score (>80% preferred)
   - Select highest-ranked template
   - Generate customization requirements (<20% modification)

5. **Optimization Application**
   - Apply learning engine optimizations
   - Integrate user feedback patterns
   - Generate execution package

#### Template Customization Thresholds

- **<10% customization**: Use template as-is
- **10-20% customization**: Apply parameterization
- **20-30% customization**: Use template + custom components
- **>30% customization**: Create new template or manual process

#### Continuous Learning Integration

The **Workflow Learning Coordinator** continuously improves selection:

- **Success Metrics Tracking**: Completion rates, time savings, quality scores
- **Pattern Learning**: Identifies successful template adaptations
- **User Feedback Integration**: Incorporates stakeholder preferences
- **Performance Optimization**: Refines selection algorithms based on outcomes
