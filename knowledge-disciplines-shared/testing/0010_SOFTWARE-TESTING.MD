# Software Testing Project Procurement Typology

## Sector Overview
Software testing procurement covers test management tools, automation frameworks, load testing infrastructure, device labs, and professional testing services. This is a shared discipline applicable across all project types requiring software QA.

## Common PO Types

| PO Category | Typical Goods | Key Specifications | Lead Times |
|-------------|---------------|-------------------|------------|
| Test Management Tools | JIRA/Zephyr, TestRail, qTest, ALM licenses | User count, project limits, API access, reporting | 1-4 weeks |
| Automation Frameworks | Selenium, Playwright, Cypress, Appium licenses | Language support, browser coverage, CI integration | 1-4 weeks |
| Load Testing Hardware | Dedicated load generators, network emulators, APM appliances | VUser capacity, bandwidth shaping, protocol support | 4-12 weeks |
| Device Labs | Physical devices, device racks, SIM management | OS version coverage, device types, screen sizes | 2-8 weeks |

## Common WO Types

| WO Category | Typical Services | Compliance Requirements |
|-------------|-----------------|------------------------|
| Manual Testing | Functional, regression, UAT, exploratory testing | Test case coverage, defect tracking, test sign-off |
| Automation Engineering | Framework development, script maintenance, CI/CD integration | Coding standards, coverage targets, execution reporting |
| Performance Testing | Load, stress, endurance, scalability testing | SLA thresholds, concurrency targets, response time SLAs |
| Security Testing | Penetration testing, vulnerability assessment, SAST/DAST | OWASP Top 10, compliance standards, remediation tracking |
| UAT Facilitation | Test script preparation, business user coordination, sign-off | Acceptance criteria traceability, defect triage, UAT completion cert |

## Key Distinctions
### Goods (PO Items)
- Software licenses with concurrent user or named user models
- Hardware with capacity planning for load generation requirements
- Device management with inventory and OS version tracking

### Services (WO Items)
- Testing with documented test plans, scripts, and results
- Security testing with find classification and remediation tracking
- Performance testing with baseline comparisons and bottleneck analysis

## Project-Specific Considerations
- **Web Applications:** Cross-browser testing, responsive design testing, accessibility compliance
- **Mobile Applications:** Device fragmentation, OS version coverage, network conditions simulation
- **Enterprise Systems:** Integration testing, data migration testing, batch processing validation
- **Real-Time Systems:** Latency testing, concurrency testing, failover testing
