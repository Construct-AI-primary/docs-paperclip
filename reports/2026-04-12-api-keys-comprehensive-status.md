## Technical Implementation

### Database Schema

```sql
-- Agent API Keys Table Structure
CREATE TABLE agent_api_keys (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  agent_id uuid NOT NULL REFERENCES agents(id) ON DELETE CASCADE,
  company_id uuid NOT NULL REFERENCES companies(id) ON DELETE CASCADE,
  name text NOT NULL,
  key_hash text NOT NULL UNIQUE,
  last_used_at timestamptz,
  revoked_at timestamptz,           -- NULL = active, NOT NULL = revoked
  created_at timestamptz NOT NULL DEFAULT NOW()
);

-- Active Keys: WHERE revoked_at IS NULL
-- Revoked Keys: WHERE revoked_at IS NOT NULL
```

### Key Management Principles

1. **One Active Key Per Agent:** Each agent has exactly one active API key
2. **Soft Delete:** Revoked keys retained for audit purposes
3. **SHA-256 Hashing:** All keys stored as secure hashes
4. **Timestamp Tracking:** Created and revoked timestamps for full audit trail

## Verification Queries

To verify current state in your Supabase database:

```sql
-- Overall Coverage Summary
SELECT 
  COUNT(DISTINCT a.id) as total_active_agents,
  COUNT(DISTINCT ak.agent_id) as agents_with_keys,
  COUNT(DISTINCT ak.id) as total_active_keys,
  ROUND(100.0 * COUNT(DISTINCT ak.agent_id) / COUNT(DISTINCT a.id), 2) as coverage_percent
FROM agents a
LEFT JOIN agent_api_keys ak ON a.id = ak.agent_id AND ak.revoked_at IS NULL
WHERE a.status = 'active';

-- Company-by-Company Breakdown
SELECT 
  c.name as company,
  COUNT(DISTINCT a.id) as active_agents,
  COUNT(DISTINCT ak.id) as active_keys,
  COUNT(DISTINCT ak.id) FILTER (WHERE ak.revoked_at IS NOT NULL) as revoked_keys
FROM companies c
LEFT JOIN agents a ON c.id = a.company_id AND a.status = 'active'
LEFT JOIN agent_api_keys ak ON a.id = ak.agent_id
GROUP BY c.name
ORDER BY c.name;
```
